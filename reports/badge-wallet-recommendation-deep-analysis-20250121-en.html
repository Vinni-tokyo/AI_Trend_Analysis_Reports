Executive Summary

This report provides an in-depth analysis of the technical architecture and algorithms of five major digital badge/credential wallet platforms equipped with AI-powered recommendation systems. Each platform implements unique approaches to skills matching and career path recommendations, utilizing various technologies including graph technology, machine learning, and natural language processing.

  • Credly: Knowledge graph and NLP based on Pearson Ontology, mapping 5,000+ occupations and 200,000+ job titles
  • Territorium: CLR/LER integrated AI career pathways with automatic Top 5 essential skills matching
  • Workday: Graph technology and ML with 5B+ skills data enabling real-time adaptive recommendations
  • TalentPass: LER standard-based design focusing on verifiable credential interoperability
  • LearnCard: Open-source decentralized wallet connecting skills networks through Skills Bridge

1. Credly (Pearson) Recommendation System Deep Dive

Credly

Pearson Ontology-based Knowledge Graph

AI Recommendation Engine of the World's Largest Digital Badge Platform

1.1 Technical Architecture

Pearson Ontology Structure

• Knowledge Graph: Skill-occupation-credential relationship mapping
• 5,000+ Occupations: Global occupation classification system
• 200,000+ Job Titles: Normalized job titles
• Real-time Updates: Based on millions of job postings

Machine Learning Pipeline

• NLP Processing: Job posting text analysis
• Skill Inference: Similar/implied skill connections
• Deduplication: Automatic skill normalization
• Trend Detection: Skills demand change tracking

1.2 Recommendation Algorithm Details

Credly Recommendation Logic Flow:
1. Analyze user's badge/skill profile
2. Identify related skill clusters in Pearson Ontology
3. Match with labor market trends via NLP
4. Skill gap analysis → Calculate recommended badge priority
5. Present personalized pathways based on career goals

1.3 Key Features

Labor Market Insights Demand trends, salary data, regional distribution for specific skills/occupations
Credential Recommendations Suggest next badges/credentials based on current skills
Strategic Workforce Planning Organization-level skill gap analysis and talent development pathways (B2B)
Talent Directory Two-way matching between badge holders and hiring opportunities

2. Territorium LifeJourney Recommendation System Deep Dive

Territorium

AI-Powered CLR/LER Integration Platform

Comprehensive Learner Record-based Recommendations for Education-Employment Connection

2.1 Technical Architecture

CLR (Comprehensive Learner Record)

• 1EdTech Certified: Standard compliance verified
• Competency Mapping: Auto-convert learning experiences to skills
• LMS Integration: Canvas, Blackboard, Moodle connected
• Beyond Grades: Includes informal learning

AI Career Pathways

• Top 5 Skills Matching: Extract essential competencies per role
• Gap Analysis: Compare owned vs required skills
• Path Generation: AI-based learning path recommendations
• Real-time Matching: LinkedIn, Indeed integration

2.2 Recommendation Algorithm Details

LifeJourney AI Matching Process:
1. Extract verified competencies/skills from CLR data
2. Analyze required skill profile for target positions
3. ML model calculates Top 5 essential skills priority
4. Compute matching score between learner profile and job requirements
5. Recommend learning experiences based on skill gaps (badges, courses, credentials)

2.3 Differentiating Factors

Skills Digital Wallet CLR/LER integrated digital wallet for lifelong learning record management
Stackable Credentials Accumulation and linking of micro-credentials to higher qualifications
K-12 to Workforce Support for entire lifecycle from elementary to employment
Assessment Integration Automatic conversion of competency assessment results to skills

3. Workday Skills Cloud + WayTo Recommendation System Deep Dive

Workday

Graph Technology-based Skills Cloud

Enterprise Solution Modeling 55,000 Skills Relationships as Graph

3.1 Technical Architecture

Skills Cloud Structure

• Graph DB: Store skill entities and relationships
• 55,000 Skills: Refined from 200M raw data
• O*NET Seed: Based on US government occupation data
• 5B+ Usage: Customer real usage data

ML Pipeline

• Skill Inference: Auto-derive skills from work activities
• Similarity Calculation: Measure distance/relationships between skills
• Real-time Adaptation: Instant recommendation updates on new skill learning
• Annotation: External data normalization

3.2 Recommendation Algorithm Details

Workday Adaptive Recommendation System:
1. Employee Profile: Collect owned skills, career history, learning records
2. Graph Exploration: Explore related skill clusters and pathways
3. Career Hub: Skills-based role/opportunity matching
4. Talent Marketplace: Project/gig opportunity recommendations
5. Feedback Loop: Instantly update recommendations reflecting learning/experience

3.3 WayTo Wallet Features

Blockchain Credentials Issue and store blockchain-based verifiable credentials
Portable Identity User-owned portable digital identity
Selective Disclosure User controls what to share and with whom
Skills Interoperability Bidirectional skills data exchange with external systems

3.4 Skills Software Alliance Partners

Workday provides data interoperability through pre-mapped skills ontologies with the following partners:

  • Aon: Talent assessment and skills data integration
  • Degreed: Learning experience platform skills connection
  • SkyHive: Labor market skills intelligence integration

LER Interoperability

Technology Trends and Outlook

1. Rise of Graph Technology

Both Credly and Workday adopt knowledge graph technology as core. Modeling relationships between skills enables more accurate inference and recommendations.

2. Real-time Adaptive Recommendations

Workday's approach of "all recommendations instantly updating when learning new skills" is spreading as an industry standard.

3. Open Standards vs Proprietary Ontology

Proprietary ontologies of Credly/Workday coexist with LearnCard's open approach. The market increasingly demands interoperability.

4. Accelerating CLR/LER Integration

Territorium's CLR+LER approach emerges as a best practice for education-employment connection. 1EdTech standard adoption expanding.

Future Development Directions

  • AI Recommendation Advancement: Expected evolution toward LLM-based conversational career coaching
  • Decentralized Identity Standardization: DID/VC-based self-sovereign identity moving to mainstream
  • Skills Passport Integration: Linking with government-led initiatives like EU Digital Identity Wallet
  • Real-time Labor Market Feed: Real-time synchronization of job posting data with skills ontology

Conclusions and Recommendations

Platform Selection Guide by Use Case

  • Enterprise Talent Management: Workday Skills Cloud - Enterprise-grade scalability and HCM integration
  • Global Credential Network: Credly - Largest badge ecosystem and hiring connection
  • Educational Institution Learning Records: Territorium LifeJourney - CLR/LER standards and LMS integration
  • Decentralized Self-Sovereign Identity: TalentPass/LearnCard - VC/DID-based privacy-centric
  • Open-Source Customization: LearnCard - Developer-friendly, plugin extensible
