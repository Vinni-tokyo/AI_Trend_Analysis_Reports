Top 5 Trends

1. 1EdTech Edu-API: Next-Generation SIS API Standard for Higher Education

Current Status: In October 2025, Oracle PeopleSoft Campus Solutions became the first student information system to achieve 1EdTech Edu-API certification. Edu-API is a next-generation SIS API standard developed for higher education institutions, building on the success of OneRoster in K-12 education. It dramatically reduces implementation time and costs associated with custom integrations, enabling institutions to securely exchange critical data across their application ecosystems using standardized methods.

2. Enhanced Security Standards: Ransomware Defense and Secure Educational Data Exchange

Current Status: In the first half of 2025, ransomware attacks against schools and universities rose 23% year-over-year. In response to this threat, 1EdTech (formerly IMS Global) has implemented OAuth 2.0, OIDC, and JWT-based security frameworks across interoperability standards including LTI, Edu-API, and OneRoster. Adoption of interoperability standards is now recognized not merely as a convenience improvement but as a critical cyber defense measure.

5. CASE Standard: Machine-Readable Competency-Based Education

Current Status: CASE (Competencies and Academic Standards Exchange) is a 1EdTech standard that uniquely identifies learning standards and competencies with GUIDs (Globally Unique Identifiers) and exchanges them in machine-readable formats. Traditionally, learning standards were published in inefficient formats like PDFs, HTML, and spreadsheets, but CASE enables accurate and consistent sharing of learning outcome data across LMS, assessment tools, curriculum management systems, and digital credentialing platforms.

Technical Specifications

‚Ä¢ GUID Identifiers: Assigns unique IDs to each learning standard and competency
‚Ä¢ Machine-Readable Format: Exchanged via APIs in JSON-LD format
‚Ä¢ Hierarchical Support: Accurately represents parent-child relationships of learning objectives

Interoperability Realization

‚Ä¢ Assessment Tools: Links test questions to learning standards
‚Ä¢ Credentials: Connects digital badges with skill standards

Key Terminology

LTI (Learning Tools Interoperability) - Standard enabling seamless connection between LMS and external tools (educational apps). Allows one-click access to tools.
OneRoster - Standard for sharing class rosters and grade data between SIS (Student Information Systems) and LMS. Supports CSV format and REST API format.
xAPI (Experience API / Tin Can API) - Standard capable of collecting and analyzing all learning data, including activities outside LMS (mobile apps, VR, field training). SCORM successor.
LRS (Learning Record Store) - Central database managing xAPI statements. Collects and analyzes learning data from all platforms.
Ed-Fi Data Standard - Common language for K-12 educational data. Standard integrating data previously dispersed across individual systems: attendance, grades, test scores.
Edu-API - SIS (Student Information System) API standard for higher education institutions. Specification extending OneRoster for universities and colleges.
CASE (Competencies and Academic Standards Exchange) - Standard identifying learning standards and competencies with GUIDs (Globally Unique Identifiers) for exchange in machine-readable formats.
OAuth 2.0 / OIDC / JWT - Modern authentication and authorization standard technologies. Adopted by LTI 1.3 and Edu-API to enable secure data exchange.
