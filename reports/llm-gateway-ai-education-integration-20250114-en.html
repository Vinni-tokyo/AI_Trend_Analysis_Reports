Key Terminology

LLM Gateway - Infrastructure that provides unified access to multiple large language models (LLMs), offering cost optimization, security, routing, and observability. Acts as an intermediary point for enterprises to efficiently utilize multiple AI models (OpenAI, Anthropic, Google, etc.).
Intelligent Routing - Technology that analyzes AI request content and complexity to automatically select the optimal LLM. Balances cost, latency, and quality while distributing tasks to appropriate models, achieving 40-85% cost reduction.
Adaptive Learning - Learning system that analyzes student performance and comprehension in real-time to automatically adjust difficulty and content. AI identifies individual strengths and weaknesses, providing optimal learning paths to maximize learning effectiveness.
Local LLM - Small-scale LLMs (3B-7B parameters) operating on-premises (company or school servers) rather than in the cloud. Enables AI functionality while protecting student data privacy without external transmission.
LLM Sprawl - Phenomenon where numerous LLMs are introduced haphazardly within enterprises or organizations, causing cost inflation, security risks, and management complexity. Centralized management through LLM Gateways is the solution.
Semantic Caching - Technology that stores results from past similar AI requests and reuses them when similar questions arise. Portkey can reduce 90% of API calls with this feature.
LLMOps (LLM Operations) - Practices for operating, managing, and monitoring large language models. An evolution of machine learning's MLOps, including cost tracking, performance monitoring, and model version management.
