Key Terminology

Vibe Coding - Development methodology proposed by Andrej Karpathy in February 2025, where developers use natural language prompts to guide AI in code generation. Represents a new paradigm of intuitive development based on "feeling" rather than detailed specifications
Disposable Software - Software created quickly, used immediately, and discarded later. Design philosophy prioritizing immediate problem-solving and agility over long-term maintenance or extensibility. Sketching ideas in code like writers sketch in words
AI-Enhanced IDE - Next-generation integrated development environments like Cursor and Windsurf that integrate AI-powered code generation, prediction, execution, and debugging. Beyond traditional code completion, can understand developer intent and generate entire code segments
AI Governance Structure - Framework defining responsibilities, authorities, and decision-making processes for AI utilization within organizations. Various models exist (CTO-led, COO oversight, GM-driven) depending on company culture and strategy
Workslop - Phenomenon where inappropriate AI tool usage creates more work for colleagues regardless of necessity. Newly coined term by researchers describing an unproductive pattern in the AI era
Prompt-Based Development - Development approach where natural language instructions (prompts) guide AI to generate software instead of writing code directly. Representative tools include v0, Lovable, and Bolt.new
Hybrid Development Approach - Development methodology combining rapid prototyping through disposable apps with traditional maintainable codebases. Practical strategy balancing experimentation with stability
